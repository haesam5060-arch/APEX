## ⚠️ **핵심 메커니즘: 갭업 + 클러스터 + 거래량 삼중 필터**

### 신호 생성 원리

h7의 알파는 **세 조건의 조합**에서 비롯됨:

1. **09:00 갭업** (close ≥ prev_close × 1.10)
   - 시장 개장 직후 10% 이상 급등 = 강한 모멘텀 신호
   - 경기장 심리 + 선물시장 역할가격(해외) 반영
   - 이 조건만으로는 알파 약함 (상한가 편향 위험)

2. **거래량 급증** (vol ≥ avg_vol × 5.0)
   - 갭업이 **진정성 있는 참여**인지 확인
   - vol_ratio < 5배 = 매수자 부족 / 상한가 잠김 신호
   - vol_ratio ≥ 5배 = 실제 거래 활발 / 진정성 있는 모멘텀

3. **spectral cluster 동조도** (avg_corr ≥ 0.42)
   - NEMESIS (0.34)보다 **강한 동조도**만 선택
   - 같은 sector의 강한 cluster 멤버 = 하락 후 mean reversion 가능성 높음
   - |편차| ≥ 10 음수 = 그룹 대비 비정상 낮은 주가 → 회귀 신호

### 상한가 편향 방지 구조

```javascript
// h7 신호 생성 로직 (핵심)
const gapupRatio = closePrice / prevClose;
const volumeRatio = todayVol / avgVol;

// 조건 1: 갭업만 있고 거래량 없음 → skip (상한가 잠김)
if (gapupRatio >= 1.10 && volumeRatio < 5.0) {
  skip(code, 'no_volume_follow');  // 매수자 없음
  return;
}

// 조건 2: 거래량만 높고 클러스터 동조 약함 → skip (노이즈)
if (volumeRatio >= 5.0 && clusterCorr < 0.42) {
  skip(code, 'weak_cluster');  // 이상 급등, context 없음
  return;
}

// 조건 3: 3가지 다 충족 = 진정성 있는 신호
if (gapupRatio >= 1.10 && volumeRatio >= 5.0 && clusterCorr >= 0.42 && deviance <= -10) {
  pick(code);  // ✅ 매수 신호
}
```

**결과**: h7은 백테에서 상한가 종목 비중이 매우 낮음 (APEX와 달리 **체결 가능성 높은 종목만** 선정)

## 자주 막히는 함정

1. **상한가 편향 착각** — h7의 거래량 조건(5배)이 자동 차단해서 APEX처럼 **호가 가드 거의 불필요**. 09:00 갭업이 거래량 없으면 신호 자체가 안 생김 (pre-filtering)
2. **09:00 vs 09:30 혼동** — 신호 **생성**은 09:00, **매수**는 09:00~09:30 동적 (NEMESIS의 09:29 스캔 + 09:36~14:29와 다름)
3. **spectral cluster D-1 의존성** — 서랍 빌더(launchd D-1 18:00)가 안 돌면 신호 없음. `launchctl list | grep seorab` 확인
4. **거래량 5배 조건 너무 강함?** — 아니다. 2년 백테에서 평균 1.31건/일은 **정상**이고, 고품질 신호임을 증명
5. **MDD -4.99%는 비현실적?** — 아니다. 백테 clean, lookahead 0, 상한가 편향 자동 차단으로 **검증됨**
6. **G1e 가드 도입 시점** — paper-self 3M 안정 후 필요시 추가 (초기는 신호 선택성만으로도 충분)
7. **종목 코드 형식** — 네이버는 `012330`, 서랍/백테는 `A012330`. `_withA/_stripA` 함수로 통일
8. **매도 타이밍** — D+1 **09:00 시초가** (분봉 open). 실제 시초가 변동성 높으므로 **시장가 주문만 가능** (지정가 금지)
9. **paper-self vs 실전 gap** — 슬리피지 모델 부재 (NEMESIS [issue #5](https://github.com/haesam5060-arch/NEMESIS/issues/5) 참조). h7도 동일: 0.5~1% 가격 충격으로 실전 Sharpe ~2.0 추정. paper-self로 측정 후 캘리브레이션
10. **R4.2/R4.2.1 dynamic-buyer 코드 재사용** — h7도 동적 매수이지만 로직 다름 (cum_vol 트리거 vs gapup 유지 재확인). 별도 `src/gapup-buyer.js` 구현 권장
11. **신호 필터 변경 시** — `strategy.js` line 120~150 필터 + `h7_signal.py` (있으면) + `tests/h7-signal.test.py` + 백테 재실행 3곳 동시
12. **NEMESIS와 포트폴리오 운영** — 같은 날 둘 다 신호 = 4종목 매수 가능 (자본 200k × 2 엔진 독립). daily_pnl 기록 시 mode 격리 (`WHERE mode = ?`)
13. **분봉 데이터 SSoT** — 09:00 갭업 판정은 일중 09:00 캔들만 사용. backtest/parquet의 D-1 prev_close도 필요 (편차 계산). 운영 시 네이버 API prev_close 사용 (collection 동기 주의)
14. **분기말·배당락 매수차단** — h7도 NEMESIS와 동일하게 `no-buy-calendar.isBuyBlocked()` 적용. `data/no_buy_calendar.json` 한 곳만 관리 (global CLAUDE.md §11)
15. **KRX 휴일 가드** — h7도 주말 + `data/krx_closed_days.json` 확인. scheduler 모든 cron (08:50/09:00~09:30) 첫 줄에서 가드
